# Contributing

## Development Principles

- Keep changes small and focused.
- One feature per commit.
- Test every change before committing.
- Document significant changes.
- Preserve backward compatibility where practical.

---

## Coding Standards

- Use Bash best practices.
- Quote variables.
- Prefer functions over duplicated code.
- Keep functions short and readable.
- Avoid hard-coded paths whenever possible.

---

## Git Workflow

1. Create a feature.
2. Test it.
3. Commit it.
4. Continue.

Example commit messages:

- Add dependency detection
- Refactor Java compatibility
- Improve Android tooling checks
- Fix installer logging

---

## Testing Checklist

Before every commit:

- Script syntax check
- Functional test
- Dependency verification
- Review terminal output
